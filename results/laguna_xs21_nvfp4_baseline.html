⚠ Host Freeze Root Cause & Fix

Root cause: FlashInfer JIT compilation of 97 CUTLASS MoE kernel objects at runtime with unbounded parallelism (ninja defaults to all 20 CPU cores). Each nvcc process consumes 8–16 GiB RAM → 20 × ~10 GiB = ~200 GiB RAM demand on 121 GiB system → OOM → host freeze → hard reboot.
Fix applied: MAX_JOBS=6 + FLASHINFER_NVCC_THREADS=2 baked into entrypoint.sh. JIT compiles with 6 parallel nvcc × 2 threads each — peak ~97 GiB used, host survives. JIT cache persisted (172 MB) to avoid recompilation on restart.
Why this happens: Laguna's compressed-tensors nvfp4-pack format routes to FLASHINFER_CUTLASS which JIT-compiles 97 SM121-specific MoE kernels on first run.
